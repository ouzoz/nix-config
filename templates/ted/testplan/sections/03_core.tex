\section{Features To Be Tested}
\label{sec:features}
% -----------------------------------------------------------------------------

This section lists the implemented product features that are covered by current testing
activities. The emphasis is on features that already have executable verification paths,
either through automated commands or through manual smoke procedures supported by the
application.

\subsection{Feature Groups}

\subsubsection{Configuration and Runtime}

\begin{itemize}[leftmargin=*]
	\item \textbf{Config and CLI.} Validation of numeric bounds, handling of invalid
	      overrides, experiment listing, experiment selection, runtime naming, headless mode,
	      and CPU-only mode.\\
	      \textbf{Current verification mode:} unit, integration, and manual testing.

	\item \textbf{Application startup.} Construction of simulation and evolution managers,
	      optional visualization setup, signal handling, and final report logging.\\
	      \textbf{Current verification mode:} integration and manual system testing.

	\item \textbf{Output generation.} Creation of run directory, configuration snapshot,
	      metrics CSV, species CSV, genome JSON, and final flush at shutdown.\\
	      \textbf{Current verification mode:} integration and manual artifact inspection.

	\item \textbf{GPU fallback path.} Runtime execution with optional GPU enablement and
	      explicit \texttt{--no-gpu} fallback to CPU.\\
	      \textbf{Current verification mode:} manual system and performance testing.
\end{itemize}

\subsubsection{Core Algorithms}

\begin{itemize}[leftmargin=*]
	\item \textbf{Evolution correctness.} Innovation numbering, mutation behavior,
	      crossover behavior, species compatibility, network activation, and JSON round-trip
	      behavior. Current verification mode: automated unit tests.

	\item \textbf{Simulation sensing.} Nearest-entity delta encoding, missing-target
	      sentinels, and local density values in sensor vectors. Current verification mode:
	      automated unit tests.
\end{itemize}

\subsubsection{Tooling and Platform Support}

\begin{itemize}[leftmargin=*]
	\item \textbf{Cross-platform build stability.} Build and test execution on Linux and
	      Windows in Debug and Release configurations. Current verification mode: GitHub Actions CI.

	\item \textbf{Analysis reporting.} Discovery of completed runs and generation of a
	      self-contained HTML analysis report. Current verification mode: integration testing.

	\item \textbf{Profiler reporting.} Capture of scoped timing data, generation of
	      profiler JSON, and HTML performance report output. Current verification mode:
	      performance testing workflow.
\end{itemize}

\subsubsection{Visualization and Interaction}

\begin{itemize}[leftmargin=*]
	\item \textbf{Visualization usability.} Launch, pause or resume, stepping, zoom,
	      camera motion, agent selection, and graceful exit. Current verification mode: manual
	      acceptance testing.
\end{itemize}

\subsection{Features Explicitly Not Covered by Current Tests}

Current tests do not attempt to prove the following because the supporting infrastructure is
not present in the repository:

\begin{itemize}[nosep]
	\item usability studies with external non-team users,
	\item load balancing or distributed experiment execution,
	\item long-duration soak testing across the full 330-run matrix on every target platform,
	\item correctness claims for third-party dependencies beyond successful integration,
	\item GUI regression capture through screenshot-diff tooling.
\end{itemize}
