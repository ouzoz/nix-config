\subsection{Risks and Mitigations}

\begin{table}[H]
	\centering
	\caption{Key testing risks}
	\label{tab:risks}
	\begin{tabularx}{\textwidth}{@{}>{\ttfamily\small}lX X@{}}
		\toprule
		\textbf{Risk}                   & \textbf{Impact}                                                                          & \textbf{Mitigation}                                                                                      \\
		\midrule
		CUDA hardware unavailable       & GPU-specific performance and runtime checks cannot be executed on every machine.         & Keep CPU fallback test coverage active and run GPU checks only on the available CUDA-capable machine.    \\
		GUI requires display access     & GUI smoke tests cannot run inside fully headless automation.                             & Keep GUI checks manual and rely on headless automated paths for CI.                                      \\
		Cross-platform dependency drift & Linux and Windows builds may diverge because of toolchain or dependency changes.         & Continue running both platforms in CI for pull requests and release workflows.                           \\
		Large experiment runtime cost   & Full-scale performance or experiment runs may exceed available time before the deadline. & Use shorter smoke runs for test verification and reserve long runs for analysis tasks when time permits. \\
		Small manual tester pool        & Internal acceptance and beta coverage are limited to three team members.                 & Use a repeatable smoke checklist and ensure all three team members run the same scenarios.               \\
		\bottomrule
	\end{tabularx}
\end{table}

\subsection{Deliverables}

The implemented test process produces the following deliverables:

\begin{itemize}[nosep]
	\item automated test evidence through local CTest runs and GitHub Actions logs,
	\item generated runtime artifacts for experiment, analysis, and profiler flows.
\end{itemize}
