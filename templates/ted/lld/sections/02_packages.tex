\section{Package Overview}
\label{sec:packages}
% ─────────────────────────────────────────────────────────────────────────────

MoonAI is structured as six CMake static libraries under \texttt{src/}.  Each library
corresponds to one architectural subsystem with a single, well-defined responsibility.
Figure~\ref{fig:pkg} shows the package dependency graph.

\textbf{Key classes by package:}
\begin{itemize}[nosep]
  \item \texttt{moonai.core}: Vec2, SimulationConfig, Random, CLIArgs, BoundaryMode
  \item \texttt{moonai.simulation}: Agent, Predator, Prey, Environment, SpatialGrid, Physics, SensorInput, SimulationManager
  \item \texttt{moonai.evolution}: Genome, NeuralNetwork, Species, InnovationTracker, Mutation, Crossover, EvolutionManager
  \item \texttt{moonai.visualization}: VisualizationManager, Renderer, UIOverlay, OverlayStats
  \item \texttt{moonai.data}: Logger, MetricsCollector, GenerationMetrics
  \item \texttt{moonai.gpu}: GpuBatch, GpuNetDesc, GpuAgentStats, GpuNetworkData
\end{itemize}

\subsection{Dependency Rules}

\begin{itemize}
  \item \textbf{\texttt{moonai.core}} has no internal dependencies.  It provides
    primitive types (\texttt{Vec2}, \texttt{AgentId}), configuration (\texttt{SimulationConfig}),
    and the seeded RNG (\texttt{Random}) consumed by every other package.
  \item \textbf{\texttt{moonai.gpu}} depends only on \texttt{gpu\_types.hpp} from
    \texttt{moonai.core}.  It has no link-time dependency on \texttt{moonai.evolution},
    which prevents circular dependencies; the host-side packing of GPU network data
    lives in \texttt{EvolutionManager}.
  \item \textbf{\texttt{moonai.simulation}} depends on \texttt{moonai.core} only.
    The simulation engine is deliberately unaware of NEAT to allow independent testing.
  \item \textbf{\texttt{moonai.evolution}} depends on \texttt{moonai.core} and
    \texttt{moonai.simulation} (for \texttt{SimulationManager} in
    \texttt{evaluate\_generation}).
  \item \textbf{\texttt{moonai.visualization}} and \textbf{\texttt{moonai.data}} are
    leaf consumers: they depend on all domain packages but nothing depends on them.
\end{itemize}
