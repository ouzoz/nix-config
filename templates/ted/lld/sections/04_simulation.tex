\section{Simulation Package (\texttt{src/simulation/})}
\label{sec:simulation}
% ─────────────────────────────────────────────────────────────────────────────

The simulation package models the physical world: agents, environment, food, spatial
acceleration, sensor building, and attack processing.  It is intentionally unaware of
NEAT or neural networks.

\subsection{\texttt{Predator} : \texttt{Agent}}

\begin{lstlisting}
class Predator : public Agent {
public:
    Predator(AgentId id, Vec2 position, float speed,
             float vision_range, float energy, float attack_range);
    void update(float dt) override;
    float attack_range() const;
private:
    float attack_range_;
};
\end{lstlisting}

\texttt{update()} is a no-op stub; direction is set externally via
\texttt{SimulationManager::apply\_action()}.  \texttt{attack\_range\_} is the kill radius
checked by \texttt{Physics::process\_attacks()}.

\subsection{\texttt{SensorInput} (struct)}

The 15-element observation vector produced by \texttt{Physics::build\_sensors()} and fed
as input to each agent's neural network.

\begin{table}[H]
\centering\small
\caption{\texttt{SensorInput} fields}
\begin{tabularx}{\textwidth}{@{}lX@{}}
\toprule
\textbf{Field} & \textbf{Description} \\
\midrule

\texttt{nearest\_predator\_angle}   & $[-1, 1]$; $\mathit{atan2}(\Delta y, \Delta x)/\pi$ \\
\texttt{nearest\_prey\_dist}        & $[-1, 1]$; same encoding as predator dist \\
\texttt{nearest\_prey\_angle}       & $[-1, 1]$; same encoding as predator angle \\

\texttt{nearest\_food\_angle}       & $[-1, 1]$; direction to nearest food \\
\texttt{energy\_level}              & $[0, 1]$; \texttt{energy / initial\_energy} \\
\texttt{speed\_x}                   & $[-1, 1]$; current velocity x / max speed \\
\texttt{speed\_y}                   & $[-1, 1]$; current velocity y / max speed \\
\texttt{local\_predator\_density}   & $[0, 1]$; fraction of cells in vision range containing predators \\
\texttt{local\_prey\_density}       & $[0, 1]$; fraction of cells in vision range containing prey \\
\texttt{wall\_left}                 & $[0, 1]$; distance to left boundary / world width (Clamp mode) \\
\texttt{wall\_right}                & $[0, 1]$; distance to right boundary / world width \\

\texttt{wall\_bottom}               & $[0, 1]$; distance to bottom boundary / world height \\
\bottomrule
\end{tabularx}

\texttt{static constexpr int SIZE = 15;} — must match the number of NN input nodes.\\
\texttt{to\_vector() → vector<float>} — returns all fields in declaration order.

% ─────────────────────────────────────────────────────────────────────────────
\subsection{\texttt{Physics} (static class)}

All methods are \texttt{static}.  No instance state.

\subsubsection{\texttt{build\_sensors()}}

Queries the \texttt{SpatialGrid} and \texttt{food\_grid} for neighbors within
\texttt{vision\_range}.  Algorithm:
\begin{enumerate}[nosep]
  \item Query agent's cell and 8 neighboring cells in \texttt{SpatialGrid}.
  \item For each visible agent of opposite type, compute
    $d = \text{distance\_to}$, $\theta = \text{atan2}(\Delta y,\Delta x)/\pi$.
  \item Encode: $\mathit{dist\_enc} = 1 - d/\mathit{vision\_range}$, clamped to $[-1,1]$.
  \item Repeat for food using \texttt{food\_grid}.
  \item Fill wall sensors from \texttt{BoundaryMode} and agent position.
\end{enumerate}

\subsubsection{\texttt{process\_attacks()}}

For each alive \texttt{Predator}, queries \texttt{SpatialGrid} with radius
\texttt{attack\_range}.  Marks any overlapping alive \texttt{Prey} as dead;
increments predator kill counter; adds \texttt{energy\_gain\_from\_kill}.
Returns vector of killed \texttt{AgentId} values.

% ─────────────────────────────────────────────────────────────────────────────
\subsection{\texttt{SpatialGrid} (class)}

Hash-grid for O(1) amortized spatial queries.

\begin{lstlisting}
SpatialGrid(int world_width, int world_height, float cell_size);
\end{lstlisting}

\texttt{cell\_size} is set to \texttt{vision\_range / 2} in \texttt{SimulationManager},
ensuring a query of radius \texttt{vision\_range} covers at most a 5×5 patch of cells.

\begin{table}[H]
\centering\small
\caption{\texttt{SpatialGrid} public interface}
\begin{tabularx}{\textwidth}{@{}lX@{}}
\toprule
\textbf{Method} & \textbf{Description} \\
\midrule
\texttt{clear()} & Erase all entries (called each tick before rebuild) \\
\texttt{insert(id, position)} & Place agent in appropriate cell \\
\texttt{query(position) → vector<AgentId>} & Return all agents in the cell containing \emph{position} plus 8 neighbors \\
\texttt{query\_radius(position, radius) → vector<AgentId>} & Filter \texttt{query()} results by Euclidean distance \\
\bottomrule
\end{tabularx}
\end{table}

\textbf{Private:} \texttt{cells\_: vector<vector<Entry>>} where
\texttt{Entry = \{AgentId id; Vec2 pos;\}}.
Index computed as $\mathit{cell\_y}(y) \times \mathit{cols} + \mathit{cell\_x}(x)$.

\subsubsection{Per-Tick Sequence (internal)}
\begin{enumerate}[nosep]
  \item \texttt{rebuild\_spatial\_grid()} — clear and re-insert all alive agents.
  \item \texttt{rebuild\_food\_grid()} — same for active food items.
  \item Per agent (in \texttt{evaluate\_generation}): caller calls
    \texttt{get\_sensors}, runs NN, calls \texttt{apply\_action}.
  \item \texttt{process\_energy(dt)} — drain energy; mark zero-energy agents dead.
  \item \texttt{process\_food()} — for each alive prey, call \texttt{try\_eat\_food}.
  \item \texttt{process\_attacks()} — delegate to \texttt{Physics::process\_attacks()}.
  \item \texttt{count\_alive()} — update \texttt{alive\_predators\_} and
    \texttt{alive\_prey\_}.
  \item Tick food respawn: \texttt{environment\_.tick\_food(rng\_, respawn\_rate)}.
\end{enumerate}

\textbf{Private members:} \texttt{config\_: SimulationConfig}, \texttt{rng\_: Random},
\texttt{environment\_: Environment}, \texttt{grid\_: SpatialGrid},
\texttt{food\_grid\_: SpatialGrid}, \texttt{agents\_: vector<unique\_ptr<Agent>>},
\texttt{current\_tick\_: int}, \texttt{alive\_predators\_: int}, \texttt{alive\_prey\_: int}.
